# 第 10 章：发布与订阅（Pub/Sub）

> 本章目标：学会用 Pub/Sub 实现实时消息通信。

---

## 一、什么是发布与订阅？

发布与订阅（Publish/Subscribe，简称 Pub/Sub）是一种**消息通信模式**。

### 生活中的例子

想象一个杂志订阅：

```
杂志社（发布者） → 发行杂志
                      │
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
    读者A          读者B          读者C
   (订阅者)       (订阅者)       (订阅者)
```

- **发布者**：发送消息的人
- **频道（Channel）**：消息的通道
- **订阅者**：接收消息的人

### 技术图示

```
发布者 ──发布消息──→ 频道 "news"
                        │
              ┌─────────┼─────────┐
              ▼         ▼         ▼
          订阅者A    订阅者B    订阅者C
```

**特点：**
- 发布者不知道谁在接收（解耦）
- 所有订阅者都会收到同样的消息
- 消息不会持久化，订阅者离线时会丢失

---

## 二、基本命令

### 订阅频道

```
127.0.0.1:6379> SUBSCRIBE news
Reading messages... (press Ctrl-C to quit)
1) "subscribe"
2) "news"
3) (integer) 1
```

**解释：**
- `SUBSCRIBE`：订阅一个频道
- 执行后进入"监听模式"，等待接收消息
- 返回信息：订阅操作、频道名、当前订阅的频道数

### 发布消息

在另一个终端中：

```
127.0.0.1:6379> PUBLISH news "Redis 发布新版本"
(integer) 1
```

**解释：**
- `PUBLISH`：向频道发布消息
- `news`：频道名
- `"Redis 发布新版本"`：消息内容
- `(integer) 1`：有 1 个订阅者收到了消息

### 订阅者收到消息

回到订阅者的终端，会看到：

```
1) "message"
2) "news"
3) "Redis 发布新版本"
```

返回信息：消息类型、频道名、消息内容。

### 取消订阅

```
127.0.0.1:6379> UNSUBSCRIBE news
1) "unsubscribe"
2) "news"
3) (integer) 0
```

## 三、模式订阅

除了订阅具体频道，还可以用**通配符**订阅一类频道。

```
127.0.0.1:6379> PSUBSCRIBE user:*
Reading messages... (press Ctrl-C to quit)
```

然后在另一个终端发布：

```
127.0.0.1:6379> PUBLISH user:1001 "用户上线"
(integer) 1

127.0.0.1:6379> PUBLISH user:1002 "用户上线"
(integer) 1
```

订阅者会收到：

```
1) "pmessage"
2) "user:*"
3) "user:1001"
4) "用户上线"
```

**解释：**
- `pmessage`：模式消息
- `user:*`：匹配的模式
- `user:1001`：实际频道名
- `用户上线`：消息内容

## 四、Python 代码

### 基础示例

```python
import redis
import threading
import time

# 发布者
def publisher():
    r = redis.Redis(host='localhost', port=6379, decode_responses=True)
    time.sleep(1)  # 等待订阅者启动

    r.publish('news', '第一条新闻：Redis 发布新版本')
    time.sleep(1)
    r.publish('news', '第二条新闻：Python 3.12 发布')
    time.sleep(1)
    r.publish('news', '第三条新闻：AI 技术突破')
    time.sleep(1)
    r.publish('news', 'EXIT')  # 发送退出信号

# 订阅者
def subscriber():
    r = redis.Redis(host='localhost', port=6379, decode_responses=True)
    pubsub = r.pubsub()

    # 订阅频道
    pubsub.subscribe('news')
    print('等待消息...')

    # 监听消息
    for message in pubsub.listen():
        if message['type'] == 'message':
            data = message['data']
            print(f'收到: {data}')
            if data == 'EXIT':
                break

    # 取消订阅
    pubsub.unsubscribe('news')
    print('订阅结束')

# 运行
t1 = threading.Thread(target=publisher)
t2 = threading.Thread(target=subscriber)
t2.start()
t1.start()
t1.join()
t2.join()

# 输出:
# 等待消息...
# 收到: 第一条新闻：Redis 发布新版本
# 收到: 第二条新闻：Python 3.12 发布
# 收到: 第三条新闻：AI 技术突破
# 收到: EXIT
# 订阅结束
```

### 模式订阅

```python
import redis
import threading
import time

def publisher():
    r = redis.Redis(host='localhost', port=6379, decode_responses=True)
    time.sleep(1)

    r.publish('user:1001', '用户 1001 上线')
    time.sleep(0.5)
    r.publish('user:1002', '用户 1002 上线')
    time.sleep(0.5)
    r.publish('order:2001', '新订单创建')
    time.sleep(0.5)
    r.publish('EXIT', 'EXIT')

def subscriber():
    r = redis.Redis(host='localhost', port=6379, decode_responses=True)
    pubsub = r.pubsub()

    # 模式订阅：订阅所有以 "user:" 开头的频道
    pubsub.psubscribe('user:*')
    print('等待用户相关消息...')

    for message in pubsub.listen():
        if message['type'] == 'pmessage':
            channel = message['channel']
            data = message['data']
            print(f'频道 {channel}: {data}')
            if data == 'EXIT':
                break

    pubsub.punsubscribe('user:*')
    print('订阅结束')

t1 = threading.Thread(target=publisher)
t2 = threading.Thread(target=subscriber)
t2.start()
t1.start()
t1.join()
t2.join()

# 输出:
# 等待用户相关消息...
# 频道 user:1001: 用户 1001 上线
# 频道 user:1002: 用户 1002 上线
# 订阅结束
```

## 五、实战：实时通知系统

```python
import redis
import threading
import time
import json

class NotificationSystem:
    """实时通知系统"""

    def __init__(self):
        self.r = redis.Redis(host='localhost', port=6379, decode_responses=True)

    def send_notification(self, user_id, title, content):
        """发送通知给指定用户"""
        channel = f'notification:{user_id}'
        message = json.dumps({
            'title': title,
            'content': content,
            'time': time.strftime('%Y-%m-%d %H:%M:%S')
        }, ensure_ascii=False)
        self.r.publish(channel, message)
        print(f'通知已发送给用户 {user_id}')

    def send_broadcast(self, title, content):
        """发送广播通知（所有用户）"""
        channel = 'notification:broadcast'
        message = json.dumps({
            'title': title,
            'content': content,
            'time': time.strftime('%Y-%m-%d %H:%M:%S')
        }, ensure_ascii=False)
        self.r.publish(channel, message)
        print('广播通知已发送')

    def subscribe_user(self, user_id, callback):
        """订阅用户通知"""
        r = redis.Redis(host='localhost', port=6379, decode_responses=True)
        pubsub = r.pubsub()

        # 订阅用户专属频道和广播频道
        pubsub.subscribe(f'notification:{user_id}', 'notification:broadcast')

        for message in pubsub.listen():
            if message['type'] == 'message':
                data = json.loads(message['data'])
                callback(data)

# 使用示例
ns = NotificationSystem()

# 用户通知回调
def user_callback(data):
    print(f'\n[新通知] {data["title"]}')
    print(f'内容: {data["content"]}')
    print(f'时间: {data["time"]}')

# 启动订阅线程
t = threading.Thread(target=ns.subscribe_user, args=('user:1001', user_callback))
t.start()

time.sleep(1)

# 发送通知
ns.send_notification('user:1001', '订单发货', '您的订单 #1001 已发货')
time.sleep(1)
ns.send_notification('user:1001', '优惠券到账', '您获得了一张 8 折优惠券')
time.sleep(1)
ns.send_broadcast('系统维护', '今晚 22:00-24:00 进行系统维护')

# 输出:
# 通知已发送给用户 user:1001
#
# [新通知] 订单发货
# 内容: 您的订单 #1001 已发货
# 时间: 2024-01-01 10:00:00
#
# 通知已发送给用户 user:1001
#
# [新通知] 优惠券到账
# 内容: 您获得了一张 8 折优惠券
# 时间: 2024-01-01 10:00:01
#
# 广播通知已发送
#
# [新通知] 系统维护
# 内容: 今晚 22:00-24:00 进行系统维护
# 时间: 2024-01-01 10:00:02
```

## 六、注意事项

### 1. 消息不持久化

Pub/Sub 的消息**不会保存**。如果订阅者离线，消息就丢失了。

```
发布者发消息 → 没有订阅者 → 消息丢弃
```

如果需要持久化，应该用 **List** 或 **Stream**（后续章节）。

### 2. 消息不会重发

```
订阅者断线 → 发布者发消息 → 订阅者重连 → 断线期间的消息丢失
```

### 3. 适用场景

| 适合 | 不适合 |
|------|--------|
| 实时通知 | 需要持久化的消息 |
| 聊天室 | 不能丢失的消息 |
| 实时数据推送 | 需要历史记录的 |

## 七、本章小结

### 你需要记住的命令

| 命令 | 作用 | 示例 |
|------|------|------|
| `SUBSCRIBE` | 订阅频道 | `SUBSCRIBE channel` |
| `UNSUBSCRIBE` | 取消订阅 | `UNSUBSCRIBE channel` |
| `PUBLISH` | 发布消息 | `PUBLISH channel message` |
| `PSUBSCRIBE` | 模式订阅 | `PSUBSCRIBE pattern:*` |
| `PUNSUBSCRIBE` | 取消模式订阅 | `PUNSUBSCRIBE pattern:*` |

### 核心理解

1. Pub/Sub 是**一对多**的消息通信模式
2. 消息**不持久化**，适合实时通知
3. 订阅者离线时会丢失消息
