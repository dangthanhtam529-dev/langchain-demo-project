# 第 11 章：事务（Transaction）与管道（Pipeline）

> 本章目标：学会如何批量执行命令，并保证操作的原子性。

---

## 一、为什么需要事务？

### 一个出错的场景

假设你要转账：

```python
# 从 A 账户扣钱
r.decrby('account:A', 100)
# 给 B 账户加钱
r.incrby('account:B', 100)
```

**如果第一条命令执行后出错了呢？**
- A 扣了 100
- B 没加 100
- 钱凭空消失了！

**事务的作用：** 把多个命令打包，要么全部执行，要么全部不执行。

## 二、Redis 事务的基本命令

### MULTI：开始事务

```
127.0.0.1:6379> MULTI
OK
```

执行 `MULTI` 后，Redis 进入"事务模式"。

### 命令入队

```
127.0.0.1:6379> SET name "张三"
QUEUED

127.0.0.1:6379> SET age "25"
QUEUED

127.0.0.1:6379> SET city "北京"
QUEUED
```

**注意：** 返回 `QUEUED` 表示命令已入队，但**还没有执行**。

### EXEC：执行事务

```
127.0.0.1:6379> EXEC
1) OK
2) OK
3) OK
```

`EXEC` 会把队列中的所有命令**按顺序执行**，并返回每个命令的结果。

### DISCARD：取消事务

```
127.0.0.1:6379> MULTI
OK

127.0.0.1:6379> SET name "张三"
QUEUED

127.0.0.1:6379> DISCARD
OK
```

`DISCARD` 会清空命令队列，取消事务。

## 三、完整示例

```
127.0.0.1:6379> MULTI

127.0.0.1:6379> SET user:1:name "张三"

1) OK
```

## 四、Python 中的事务

在 Python 中，通常使用 **Pipeline** 来实现事务。

```python
import redis

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# 方式 1：使用 pipeline
pipe = r.pipeline()
pipe.multi()
pipe.set('user:1:name', '张三')
pipe.set('user:1:age', 25)
pipe.set('user:1:email', 'zhangsan@example.com')
results = pipe.execute()
print(results)  # 输出: [True, True, True]

# 方式 2：使用上下文管理器（推荐）
with r.pipeline() as pipe:
    pipe.multi()
    pipe.set('key1', 'value1')
    pipe.set('key2', 'value2')
    results = pipe.execute()
    print(results)  # 输出: [True, True]
```

## 五、管道（Pipeline）

### 什么是管道？

管道允许**一次性发送多个命令**到 Redis，然后一次性接收所有结果。

**不使用管道：**
```
客户端 → 命令1 → 等待响应 → 命令2 → 等待响应 → 命令3 → 等待响应
```

**使用管道：**
```
客户端 → [命令1, 命令2, 命令3] → 等待所有响应
```

### 性能对比

```python
import redis
import time

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# 普通方式
start = time.time()
for i in range(1000):
    r.set(f'key:{i}', f'value:{i}')
print(f'普通方式: {time.time() - start:.4f}秒')

# 管道方式
start = time.time()
pipe = r.pipeline()
for i in range(1000):
    pipe.set(f'key2:{i}', f'value:{i}')
pipe.execute()
print(f'管道方式: {time.time() - start:.4f}秒')
```

**通常管道方式会快 5-10 倍。**

## 六、WATCH：乐观锁

### 什么是乐观锁？

乐观锁假设**冲突很少发生**，先执行操作，如果检测到冲突再重试。

### 转账场景

```python
import redis

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# 初始化账户
r.set('account:A', 1000)
r.set('account:B', 1000)

def transfer(from_account, to_account, amount):
    """转账（使用 WATCH 保证安全）"""
    with r.pipeline() as pipe:
        while True:
            try:
                # 1. 监视要读取的键
                pipe.watch(from_account)

                # 2. 读取当前余额
                balance = int(r.get(from_account))

                # 3. 检查余额是否足够
                if balance < amount:
                    print('余额不足')
                    pipe.unwatch()
                    return False

                # 4. 开始事务
                pipe.multi()
                pipe.decrby(from_account, amount)
                pipe.incrby(to_account, amount)

                # 5. 执行事务
                pipe.execute()
                print(f'转账成功: {amount} 从 {from_account} 到 {to_account}')
                return True

            except redis.WatchError:
                # 如果在 WATCH 和 EXEC 之间键被其他客户端修改了，重试
                print('数据被修改，重试中...')
                continue

# 测试
transfer('account:A', 'account:B', 100)
print(f'A: {r.get("account:A")}, B: {r.get("account:B")}')
# 输出: A: 900, B: 1100
```

### WATCH 的工作原理

```
客户端 A                    Redis                    客户端 B
  │                          │                          │
  │── WATCH account:A ──────→│                          │
  │                          │                          │
  │── GET account:A ────────→│                          │
  │←── 1000 ─────────────────│                          │
  │                          │                          │
  │                          │←── SET account:A 500 ────│ (其他客户端修改)
  │                          │                          │
  │── MULTI ────────────────→│                          │
  │── DECRBY account:A 100 ─→│                          │
  │── EXEC ─────────────────→│                          │
  │←── 失败（数据已变）──────│                          │
  │                          │                          │
  │ 重试...                  │                          │
```

## 七、实战：库存扣减

```python
import redis

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

class InventoryManager:
    """库存管理"""

    def __init__(self):
        self.r = redis.Redis(host='localhost', port=6379, decode_responses=True)

    def init_stock(self, product_id, stock):
        """初始化库存"""
        self.r.set(f'stock:{product_id}', stock)
        print(f'商品 {product_id} 库存: {stock}')

    def buy(self, product_id, quantity):
        """购买商品（使用事务保证原子性）"""
        with self.r.pipeline() as pipe:
            while True:
                try:
                    # 监视库存键
                    pipe.watch(f'stock:{product_id}')

                    # 获取当前库存
                    current_stock = int(self.r.get(f'stock:{product_id}') or 0)

                    if current_stock < quantity:
                        print(f'库存不足，当前: {current_stock}')
                        pipe.unwatch()
                        return False

                    # 开始事务
                    pipe.multi()
                    pipe.decrby(f'stock:{product_id}', quantity)
                    pipe.incrby(f'sold:{product_id}', quantity)

                    # 执行
                    pipe.execute()
                    print(f'购买成功: {quantity} 件')
                    return True

                except redis.WatchError:
                    print('库存被修改，重试中...')
                    continue

    def get_stock(self, product_id):
        """查看库存"""
        return int(self.r.get(f'stock:{product_id}') or 0)

# 使用示例
im = InventoryManager()

# 初始化库存
im.init_stock('iPhone15', 10)

# 模拟并发购买
# 购买 3 件
im.buy('iPhone15', 3)
print(f'剩余库存: {im.get_stock("iPhone15")}')

# 再购买 5 件
im.buy('iPhone15', 5)
print(f'剩余库存: {im.get_stock("iPhone15")}')

# 购买 3 件（库存不足）
im.buy('iPhone15', 3)

# 输出:
# 商品 iPhone15 库存: 10
# 购买成功: 3 件
# 剩余库存: 7
# 购买成功: 5 件
# 剩余库存: 2
# 库存不足，当前: 2
```

## 八、事务的注意事项

### 1. 不支持回滚

Redis 事务和数据库事务不同：

```
MULTI
SET key1 value1
SET key2 value2  ← 这条命令语法错误
EXEC
```

**结果：** `key1` 会被设置，`key2` 会报错，但 `key1` **不会回滚**。

### 2. 语法错误在入队时检测

```
127.0.0.1:6379> MULTI
OK

127.0.0.1:6379> SET key1 value1
QUEUED

127.0.0.1:6379> INVALIDCOMMAND
(error) ERR unknown command

127.0.0.1:6379> EXEC
(error) EXECABORT Transaction discarded because of previous errors.
```

语法错误会导致整个事务被取消。

### 3. 运行时错误不会中断

```
127.0.0.1:6379> SET counter "abc"
OK

127.0.0.1:6379> MULTI
OK

127.0.0.1:6379> INCR counter
QUEUED

127.0.0.1:6379> SET name "张三"
QUEUED

127.0.0.1:6379> EXEC
1) (error) ERR value is not an integer
2) OK
```

`INCR` 失败了，但 `SET` 仍然执行了。

## 九、本章小结

### 你需要记住的命令

| 命令 | 作用 | 示例 |
|------|------|------|
| `MULTI` | 开始事务 | `MULTI` |
| `EXEC` | 执行事务 | `EXEC` |
| `DISCARD` | 取消事务 | `DISCARD` |
| `WATCH` | 监视键 | `WATCH key` |

### 核心理解

1. 事务把多个命令打包，**按顺序执行**
2. **不支持回滚**，和数据库事务不同
3. **Pipeline** 既可以实现事务，也可以批量执行命令提升性能
4. **WATCH** 实现乐观锁，适合并发场景
