# 第 3 章：连接 Redis 与第一个命令

> 本章目标：学会如何连接 Redis，并掌握最基础的存取操作。

---

## 一、连接 Redis

Redis 启动后，你需要一个"客户端"来和它对话。

### 什么是客户端？

想象你去餐厅：
- **Redis 服务器** = 厨房（负责处理和存储数据）
- **客户端** = 服务员（帮你把需求传达给厨房）

你需要通过客户端来告诉 Redis 你想做什么。

### 使用 redis-cli 连接

`redis-cli` 是 Redis 自带的命令行客户端，安装 Redis 时会自动安装。

```bash
# 连接本地 Redis（默认 localhost:6379）
redis-cli
```

连接成功后，你会看到这样的提示符：

```
127.0.0.1:6379>
```

**这个提示符的含义：**
- `127.0.0.1`：你连接的是本机
- `6379`：连接的端口号
- `>`：等待你输入命令

### 连接远程 Redis

```bash
# 连接指定主机和端口
redis-cli -h 192.168.1.100 -p 6379

# 带密码连接
redis-cli -h 127.0.0.1 -p 6379 -a your_password
```

**参数说明：**
- `-h`：指定主机地址（host）
- `-p`：指定端口（port）
- `-a`：指定密码（auth）

## 二、第一个命令：SET 和 GET

现在我们来存第一条数据，然后把它取出来。

### SET 命令：存数据

在 `redis-cli` 中输入：

```
127.0.0.1:6379> SET name "张三"
OK
```

**这行命令的含义：**
- `SET`：设置/存储
- `name`：键（key），也就是这条数据的"名字"
- `"张三"`：值（value），也就是实际要存的数据
- `OK`：Redis 回复说"存好了"

### GET 命令：取数据

```
127.0.0.1:6379> GET name
"张三"
```

**这行命令的含义：**
- `GET`：获取
- `name`：你要取的数据的名字
- `"张三"`：Redis 返回了之前存进去的值

### 图示理解

```
SET name "张三"
   │      │
   │      └── 值（value）：要存的内容
   └── 键（key）：给内容起的名字

GET name
   │
   └── 键（key）：按名字找数据
```

## 三、多存几条试试

```
127.0.0.1:6379> SET age "25"
OK

127.0.0.1:6379> SET city "北京"
OK

127.0.0.1:6379> SET email "zhangsan@example.com"
OK
```

现在 Redis 内存里有 4 条数据了：

```
┌─────────────────────────────────────┐
│            Redis 内存               │
│                                     │
│  name   →  "张三"                   │
│  age    →  "25"                     │
│  city   →  "北京"                   │
│  email  →  "zhangsan@example.com"   │
│                                     │
└─────────────────────────────────────┘
```

## 四、查看有哪些键

```
127.0.0.1:6379> KEYS *
1) "name"
2) "age"
3) "city"
4) "email"
```

**解释：**
- `KEYS`：列出键的命令
- `*`：通配符，表示"所有"
- 返回结果列出了当前 Redis 里所有的键名

### 模糊匹配

```
# 查找以 "user" 开头的键
127.0.0.1:6379> KEYS user:*

# 查找以 "name" 结尾的键
127.0.0.1:6379> KEYS *name
```

---

## 五、删除数据

```
127.0.0.1:6379> DEL name
(integer) 1
```

**解释：**
- `DEL`：删除（delete）
- `name`：要删除的键
- `(integer) 1`：成功删除了 1 个键

删除后再试试获取：

```
127.0.0.1:6379> GET name
(nil)
```

`(nil)` 表示这个键不存在或没有值。

---

## 六、检查键是否存在

```
127.0.0.1:6379> EXISTS age
(integer) 1

127.0.0.1:6379> EXISTS name
(integer) 0
```

**解释：**
- `EXISTS`：检查键是否存在
- 返回 `1`：存在
- 返回 `0`：不存在

---

## 七、查看键的类型

```
127.0.0.1:6379> TYPE age
string
```

**解释：**
- `TYPE`：查看键的数据类型
- `string`：字符串类型（Redis 最基础的类型）

---

## 八、清空所有数据

```
127.0.0.1:6379> FLUSHALL
OK
```

**警告：** 这个命令会删除 Redis 中**所有数据库**的所有数据，使用时要非常小心！

如果只想清空当前数据库：

```
127.0.0.1:6379> FLUSHDB
OK
```

## 九、用 Python 来做同样的事

命令行学会了，现在看看在 Python 代码中怎么做。

### 第一步：安装 redis 库

```bash
pip install redis
```

### 第二步：连接并操作

```python
import redis

# 连接 Redis
r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# decode_responses=True 的作用：
# Redis 默认返回 bytes 类型（如 b'张三'）
# 设置这个参数后，会自动转成字符串（'张三'）
# 对初学者来说更方便

# SET
r.set('name', '张三')

# GET
print(r.get('name'))  # 输出: 张三

# 多条数据
r.set('age', '25')
r.set('city', '北京')

# 获取所有键
print(r.keys('*'))  # 输出: ['name', 'age', 'city']

# 删除
r.delete('name')
print(r.get('name'))  # 输出: None

# 检查是否存在
print(r.exists('age'))  # 输出: 1
print(r.exists('name'))  # 输出: 0
```

## 十、本章小结

### 你需要记住的命令

| 命令 | 作用 | 示例 |
|------|------|------|
| `SET` | 存数据 | `SET key value` |
| `GET` | 取数据 | `GET key` |
| `DEL` | 删除 | `DEL key` |
| `EXISTS` | 检查是否存在 | `EXISTS key` |
| `KEYS` | 列出键 | `KEYS *` |
| `TYPE` | 查看类型 | `TYPE key` |

### 核心理解

1. Redis 是**键值存储**：通过名字（key）存取内容（value）
2. `SET` 和 `GET` 是最基础的两个命令
3. 命令行和 Python 代码的操作是一一对应的
