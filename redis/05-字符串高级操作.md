## 一、回顾：字符串是什么？

在 Redis 中，**字符串是最基本的数据类型**。

但这里的"字符串"不只是文字，它可以是：
- 普通文字：`"Hello"`
- 数字：`"100"`（Redis 会识别为数字并支持运算）
- 二进制数据：图片、序列化对象等

**限制：** 单个字符串最大 512MB。

## 二、批量操作：MSET 和 MGET

### 为什么要批量操作？

假设你要存 3 条数据：

```
# 普通方式：3 次网络往返
SET name "张三"
SET age "25"
SET city "北京"
```

```
# 批量方式：1 次网络往返
MSET name "张三" age "25" city "北京"
```

**批量操作更快**，因为只需要一次网络通信。

### 命令行操作

```
# MSET：批量设置
127.0.0.1:6379> MSET name "张三" age "25" city "北京"
OK

# MGET：批量获取
127.0.0.1:6379> MGET name age city
1) "张三"
2) "25"
3) "北京"
```

**语法说明：**
- `MSET key1 value1 key2 value2 ...`：设置多对键值
- `MGET key1 key2 key3 ...`：获取多个键的值

### Python 代码

```python
import redis

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# 批量设置
r.mset({
    'name': '张三',
    'age': '25',
    'city': '北京',
    'email': 'zhangsan@example.com'
})

# 批量获取
values = r.mget(['name', 'age', 'city'])
print(values)  # 输出: ['张三', '25', '北京']

# 注意：mget 返回的是列表，顺序和请求的键一致
# 如果某个键不存在，对应位置返回 None
values = r.mget(['name', 'nonexistent', 'city'])
print(values)  # 输出: ['张三', None, '北京']
```

## 三、GETSET：设置并返回旧值

```
127.0.0.1:6379> SET name "张三"
OK

127.0.0.1:6379> GETSET name "李四"
"张三"

127.0.0.1:6379> GET name
"李四"
```

**解释：**
- `GETSET`：先 GET（获取旧值），再 SET（设置新值）
- 返回的是**旧值**
- 常用于"读取并更新"的场景

### 应用场景：获取并重置计数器

```python
import redis

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# 假设有一个错误计数器
r.set('error_count', 5)

# 获取当前错误数，并重置为 0
old_count = r.getset('error_count', 0)
print(f'上次有 {old_count} 个错误，已重置')
# 输出: 上次有 5 个错误，已重置
```

## 四、APPEND：追加内容

```
127.0.0.1:6379> SET greeting "Hello"
OK

127.0.0.1:6379> APPEND greeting " World"
(integer) 11

127.0.0.1:6379> GET greeting
"Hello World"
```

**解释：**
- `APPEND`：在原有内容后面追加
- 返回的是追加后字符串的总长度
- 如果键不存在，等同于 `SET`

### Python 代码

```python
import redis

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# 日志追加
r.set('log', '2024-01-01 系统启动\n')
r.append('log', '2024-01-01 用户登录\n')
r.append('log', '2024-01-01 执行任务\n')

print(r.get('log'))
# 输出:
# 2024-01-01 系统启动
# 2024-01-01 用户登录
# 2024-01-01 执行任务
```

## 五、STRLEN：获取字符串长度

```
127.0.0.1:6379> SET message "Hello World"
OK

127.0.0.1:6379> STRLEN message
(integer) 11
```

**注意：** 返回的是**字节长度**，不是字符数。对于中文，一个汉字可能占多个字节。

```
127.0.0.1:6379> SET chinese "你好"
OK

127.0.0.1:6379> STRLEN chinese
(integer) 6
```

这里返回 6，因为 UTF-8 编码中，一个中文汉字占 3 个字节。

## 六、GETRANGE：截取子字符串

```
127.0.0.1:6379> SET text "Hello World"
OK

127.0.0.1:6379> GETRANGE text 0 4
"Hello"

127.0.0.1:6379> GETRANGE text 6 10
"World"

127.0.0.1:6379> GETRANGE text 0 -1
"Hello World"
```

**语法说明：**
- `GETRANGE key start end`：获取从 start 到 end 的子字符串
- 索引从 0 开始
- `-1` 表示最后一个字符
- `-2` 表示倒数第二个字符

### Python 代码

```python
import redis

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

r.set('phone', '13812345678')

# 获取前 3 位（运营商号段）
prefix = r.getrange('phone', 0, 2)
print(f'运营商: {prefix}')  # 输出: 138

# 获取后 4 位
suffix = r.getrange('phone', -4, -1)
print(f'尾号: {suffix}')  # 输出: 5678

# 中间 4 位用 * 替换（脱敏显示）
middle = r.getrange('phone', 3, 6)
masked = r.getrange('phone', 0, 2) + '****' + r.getrange('phone', 7, -1)
print(f'手机号: {masked}')  # 输出: 138****5678
```

## 七、SETRANGE：替换部分内容

```
127.0.0.1:6379> SET text "Hello World"
OK

127.0.0.1:6379> SETRANGE text 6 "Redis"
(integer) 11

127.0.0.1:6379> GET text
"Hello Redis"
```

**语法说明：**
- `SETRANGE key offset value`：从 offset 位置开始替换
- 返回替换后字符串的总长度

## 八、完整实战：用户信息缓存

```python
import redis
import json

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

class UserCache:
    """用户信息缓存"""

    def __init__(self):
        self.r = redis.Redis(host='localhost', port=6379, decode_responses=True)

    def save_user(self, user_id, user_data):
        """保存用户信息（JSON 字符串）"""
        key = f'user:{user_id}'
        # 将字典转为 JSON 字符串存入 Redis
        self.r.set(key, json.dumps(user_data, ensure_ascii=False))
        # 设置 1 小时过期
        self.r.expire(key, 3600)
        print(f'用户 {user_id} 信息已缓存')

    def get_user(self, user_id):
        """获取用户信息"""
        key = f'user:{user_id}'
        data = self.r.get(key)
        if data is None:
            print('缓存未命中，需要从数据库查询')
            return None
        # 将 JSON 字符串转回字典
        return json.loads(data)

    def update_field(self, user_id, field, value):
        """更新用户某个字段"""
        user = self.get_user(user_id)
        if user:
            user[field] = value
            self.save_user(user_id, user)
            print(f'字段 {field} 已更新')

# 使用示例
cache = UserCache()

# 保存用户
cache.save_user('1001', {
    'name': '张三',
    'age': 25,
    'email': 'zhangsan@example.com',
    'vip': True
})

# 获取用户
user = cache.get_user('1001')
print(user)
# 输出: {'name': '张三', 'age': 25, 'email': 'zhangsan@example.com', 'vip': True}

# 更新字段
cache.update_field('1001', 'age', 26)
user = cache.get_user('1001')
print(user['age'])  # 输出: 26
```

## 九、本章小结

### 你需要记住的命令

| 命令 | 作用 | 示例 |
|------|------|------|
| `MSET` | 批量设置 | `MSET k1 v1 k2 v2` |
| `MGET` | 批量获取 | `MGET k1 k2` |
| `GETSET` | 设置并返回旧值 | `GETSET key newvalue` |
| `APPEND` | 追加内容 | `APPEND key "追加"` |
| `STRLEN` | 获取长度 | `STRLEN key` |
| `GETRANGE` | 截取子串 | `GETRANGE key 0 4` |
| `SETRANGE` | 替换部分内容 | `SETRANGE key 6 "new"` |

### 核心理解

1. 批量操作（MSET/MGET）比单个操作更高效
2. `APPEND` 适合做日志追加
3. `GETRANGE` 可以做字符串截取和脱敏
4. Redis 字符串可以存 JSON，实现简单对象存储
