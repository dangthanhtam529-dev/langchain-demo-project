# 第 4 章：数字操作——计数器

> 本章目标：学会用 Redis 做数字运算，实现计数器功能。

---

## 一、为什么需要数字操作？

在实际项目中，我们经常需要统计数字：
- 文章被看了多少次？
- 商品有多少个赞？
- 网站今天有多少访客？

如果用传统数据库，每次都要：
1. 读取当前值
2. 在代码中 +1
3. 写回数据库

**三步操作，而且不是原子的**（多个用户同时操作可能出问题）。

Redis 提供了**原子操作**，一条命令搞定，而且不会出错。

---

## 二、INCR：自增 1

### 命令行操作

```
127.0.0.1:6379> SET counter 0
OK

127.0.0.1:6379> INCR counter
(integer) 1

127.0.0.1:6379> INCR counter
(integer) 2

127.0.0.1:6379> INCR counter
(integer) 3
```

**解释：**
- `INCR`：increment 的缩写，意思是"增加"
- 每次执行，值自动 +1
- 返回增加后的新值

### 重要特性：键不存在时自动从 0 开始

```
127.0.0.1:6379> INCR views
(integer) 1
```

**注意：** 即使 `views` 这个键之前不存在，Redis 也会把它当作 0，然后 +1 变成 1。

这意味着你不需要先 `SET views 0`，直接 `INCR` 就行。

---

## 三、INCRBY：自增指定数字

```
127.0.0.1:6379> SET score 100
OK

127.0.0.1:6379> INCRBY score 10
(integer) 110

127.0.0.1:6379> INCRBY score 50
(integer) 160
```

**解释：**
- `INCRBY`：increment by，意思是"增加多少"
- 第一个参数是键名
- 第二个参数是增加的数量

---

## 四、DECR 和 DECRBY：减法

```
127.0.0.1:6379> SET stock 100
OK

127.0.0.1:6379> DECR stock
(integer) 99

127.0.0.1:6379> DECRBY stock 10
(integer) 89
```

**解释：**
- `DECR`：decrement，减少 1
- `DECRBY`：decrement by，减少指定数量

---

## 五、INCRBYFLOAT：浮点数运算

```
127.0.0.1:6379> SET price 10.5
OK

127.0.0.1:6379> INCRBYFLOAT price 2.3
"12.8"

127.0.0.1:6379> INCRBYFLOAT price -1.5
"11.3"
```

**解释：**
- `INCRBYFLOAT`：增加一个浮点数（小数）
- 也可以传负数实现减法

---

## 六、Python 代码示例

```python
import redis

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# 文章阅读量统计
article_id = "article:1001"

# 初始化（可选，不初始化直接 INCR 也可以）
r.set(f'{article_id}:views', 0)

# 每次用户访问，阅读量 +1
r.incr(f'{article_id}:views')
r.incr(f'{article_id}:views')
r.incr(f'{article_id}:views')

# 查看阅读量
views = r.get(f'{article_id}:views')
print(f'文章阅读量: {views}')  # 输出: 文章阅读量: 3

# 点赞功能
r.incr(f'{article_id}:likes')
r.incr(f'{article_id}:likes')
likes = r.get(f'{article_id}:likes')
print(f'点赞数: {likes}')  # 输出: 点赞数: 2

# 批量增加
r.incrby(f'{article_id}:views', 100)
print(f'阅读量: {r.get(f"{article_id}:views")}')  # 输出: 103

# 浮点数操作（比如金额）
r.set('wallet:balance', '100.50')
r.incrbyfloat('wallet:balance', 20.00)
print(f'余额: {r.get("wallet:balance")}')  # 输出: 120.5
```

## 七、实战：网站访问统计

```python
import redis
from datetime import datetime

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

class VisitCounter:
    """网站访问统计"""

    def __init__(self):
        self.r = redis.Redis(host='localhost', port=6379, decode_responses=True)

    def record_visit(self, page):
        """记录一次页面访问"""
        today = datetime.now().strftime('%Y-%m-%d')
        key = f'visits:{today}:{page}'
        self.r.incr(key)
        print(f'页面 {page} 访问次数 +1')

    def get_visits(self, page, date=None):
        """获取页面访问次数"""
        if date is None:
            date = datetime.now().strftime('%Y-%m-%d')
        key = f'visits:{date}:{page}'
        count = self.r.get(key)
        return int(count) if count else 0

    def get_total_visits(self, date=None):
        """获取当天总访问量"""
        if date is None:
            date = datetime.now().strftime('%Y-%m-%d')
        # 获取当天所有页面的访问键
        keys = self.r.keys(f'visits:{date}:*')
        total = 0
        for key in keys:
            count = self.r.get(key)
            if count:
                total += int(count)
        return total

# 使用示例
counter = VisitCounter()

# 记录访问
counter.record_visit('首页')
counter.record_visit('首页')
counter.record_visit('关于我们')
counter.record_visit('首页')

# 查看统计
print(f'首页今天被访问了 {counter.get_visits("首页")} 次')
# 输出: 首页今天被访问了 3 次

print(f'关于我们今天被访问了 {counter.get_visits("关于我们")} 次')
# 输出: 关于我们今天被访问了 1 次
```

## 八、原子性的重要性

**什么是原子性？**

原子性意味着操作**不可分割**，要么全部完成，要么完全不执行。

### 举个出错的例子（没有原子性）

```
用户 A 读取当前值: 100
用户 B 读取当前值: 100
用户 A 计算 100 + 1 = 101，写入
用户 B 计算 100 + 1 = 101，写入
结果: 101（应该是 102，丢了 1 次）
```

### Redis INCR 的原子性

```
用户 A 执行 INCR → Redis 内部完成: 100 → 101
用户 B 执行 INCR → Redis 内部完成: 101 → 102
结果: 102（正确）
```

因为 Redis 是单线程的，命令一个接一个执行，**永远不会冲突**。

## 九、本章小结

### 你需要记住的命令

| 命令 | 作用 | 示例 |
|------|------|------|
| `INCR` | 自增 1 | `INCR counter` |
| `INCRBY` | 自增 N | `INCRBY counter 10` |
| `DECR` | 自减 1 | `DECR counter` |
| `DECRBY` | 自减 N | `DECRBY counter 5` |
| `INCRBYFLOAT` | 自增小数 | `INCRBYFLOAT price 1.5` |

### 核心理解

1. `INCR` 是最常用的命令之一，适合做计数器
2. 键不存在时，`INCR` 会自动从 0 开始
3. Redis 的数字操作是**原子的**，多线程/多用户环境下不会出错
